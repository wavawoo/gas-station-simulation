# Имитационная модель бензозаправочной станции

## Описание
Дискретно-событийная модель работы АЗС с несколькими разливочными автоматами. Моделирует поток автомобилей, очереди, обслуживание и экономические показатели.

## Функционал
- Моделирование работы АЗС в течение недели
- Генерация случайного потока автомобилей
- Управление очередями у колонок
- Учет различных марок топлива
- Расчет прибыли и статистики
- Экспорт результатов в файл

## Особенности модели
- Зависимость потока от времени суток и дня недели
- Влияние цены на интенсивность потока
- Ограничения длин очередей
- Учет запасов топлива
- Статистика обслуженных и потерянных клиентов

## Параметры моделирования
- Количество колонок (3-7)
- Максимальная длина очереди (5-9)
- Торговая наценка (5-15%)
- Время моделирования (до 30 дней)

## Архитектура и принципы ООП

### Принципы SOLID
1. **Single Responsibility Principle (SRP)**
   - `Station` - только модель станции и бизнес-логика
   - `Pump` - только модель колонки и управление очередью
   - `SimulationEngine` - только движок дискретно-событийной симуляции
   - `CarRequest` - только данные заявки
   - `Program` - только ввод параметров и запуск

2. **Open/Closed Principle (OCP)**
   - Легко добавлять новые типы событий (расширение enum `EventType`)
   - Возможность добавления новых стратегий распределения автомобилей
   - Расширяемая система типов топлива через словари

3. **Liskov Substitution Principle (LSP)**
   - Все колонки взаимозаменяемы, работают через общий интерфейс
   - Все события обрабатываются единым механизмом
   - Автомобили разных типов обрабатываются одинаково

4. **Interface Segregation Principle (ISP)**
   - Разделение ответственности между классами
   - Каждый класс имеет минимальный необходимый интерфейс
   - Отсутствие "жирных" интерфейсов

5. **Dependency Inversion Principle (DIP)**
   - Высокоуровневые модули (`SimulationEngine`) не зависят от низкоуровневых
   - Внедрение зависимостей через конструкторы
   - Использование абстракций для взаимодействия между компонентами

### Паттерны проектирования
1. **Discrete Event Simulation (DES)** - основная архитектурная парадигма
2. **Event Queue Pattern** - приоритетная очередь событий для управления временем
3. **Strategy Pattern** - различные стратегии распределения автомобилей по колонкам
4. **Factory Method** - создание автомобилей с случайными параметрами
5. **Observer Pattern** - логирование и отслеживание изменений состояния
6. **State Pattern** - состояния колонок (свободна/занята)

### Наследование и полиморфизм
- Использование перечислений (`enum`) для типов событий и состояний
- Полиморфная обработка событий через `switch` по типу
- Инкапсуляция сложной логики в методы классов
